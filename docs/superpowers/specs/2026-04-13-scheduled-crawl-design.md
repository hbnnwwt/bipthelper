# 爬虫定时任务设计

## 背景

每个分类页面的更新频率不同，需要为每个爬虫配置设置独立的自动爬取间隔。

## 设计

### 1. 数据模型

`CrawlConfig` 新增字段：

```python
auto_interval_hours: int = 0  # 自动爬取间隔（小时），0=关闭
```

### 2. 定时任务逻辑

- **调度器**：每 60 秒检查一次所有启用的配置
- **触发条件**：`auto_interval_hours > 0` 且距离上次爬取时间 >= 间隔
- **爬取模式**：增量（只爬第1页），不受 `initialized` 影响
- **并发控制**：如果正在爬取中，跳过本次

### 3. UI 设计

在配置表格中新增「自动间隔」列：

| 配置 | 大类 | 自动间隔 | 操作 |
|------|------|----------|------|
| 通知公告 | 机关教辅 | 4小时 | 禁用/启用/重置/删除 |
| 部门文件 | 机关教辅 | 24小时 | ... |
| 学术动态 | 教学科研 | 关闭 | ... |

- 点击间隔值可快速切换：关闭 / 4小时 / 8小时 / 12小时 / 24小时
- 表格列顺序调整：名称 → 大类 → 小类 → 分类 → 模式 → 自动间隔 → 进度 → 状态 → 上次爬取 → 操作

### 4. 后端实现

- 定时任务在 `main.py` 启动时后台运行
- 提供 API 修改 `auto_interval_hours`（复用现有 `/admin/configs/{id}` PUT 接口）
- 提供 API 查看「下次爬取时间」（可选）

### 5. 定时任务与手动触发的区别

| | 手动触发 | 定时任务 |
|---|---|---|
| 爬取范围 | 全量/增量（按配置状态） | 增量（只爬第1页） |
| 触发方式 | 用户点击按钮 | 定时检查自动触发 |
| 覆盖所有配置 | 是 | 是（符合间隔条件的） |
